# AWS IAM Security Implementation

## Overview

This project demonstrates how AWS Identity and Access Management (IAM) can be used to control access to cloud resources securely.

The setup focused on creating IAM users and groups, assigning permissions with policies, testing restricted access, and strengthening account security with MFA.

## Project Goal

- Create IAM users
- Create IAM groups
- Assign permissions using policies
- Apply least privilege access
- Test access restrictions
- Enable MFA for stronger security

## Tech Stack

- AWS IAM
- AWS Management Console
- JSON Policies
- MFA
- Access Control
- Security Best Practices

## What Was Implemented

- Created IAM users for controlled access
- Created IAM groups for permission management
- Attached policies to users and groups
- Tested allowed and denied actions
- Verified restricted access based on assigned permissions
- Configured MFA for stronger account protection

## Outcome

The IAM environment was configured to separate access by role, reduce unnecessary permissions, and improve account security through MFA and policy-based control.

## What I Learned

- How IAM users and groups are structured
- How policies control access to AWS services
- Why least privilege is important in cloud security
- How MFA strengthens account protection
